# Governance Panel Power BI Contract

## 1. Purpose

This document defines the semantic contract for Power BI consumption of the flat governance export generated by the pipeline.

Producer:
- src/artifacts.py
- function: save_powerbi_export

Input source artifact:
- artifacts/governance_panel_latest.json

Output artifact consumed by BI:
- artifacts/powerbi_export_latest.csv

## 2. Refresh Contract

- The CSV is regenerated whenever training pipeline or inference pipeline executes governance snapshot flow.
- The file always represents the latest governance state (single-row snapshot).
- Expected cardinality: 1 row per export cycle.

## 3. Schema Contract (CSV Columns)

| Column | Type | Nullable | Source | Description |
|---|---|---|---|---|
| generated_at | datetime (ISO string) | yes | panel.generated_at | Export generation timestamp |
| champion_model | string | yes | panel.champion_model | Active champion model filename |
| champion_mae | number | yes | panel.champion_metrics.MAE | Champion MAE metric |
| champion_rmse | number | yes | panel.champion_metrics.RMSE | Champion RMSE metric |
| promotion_status | string | yes | panel.promotion_status | Promotion report status |
| promotion_reason_code | string | yes | panel.promotion_latest_decision.reason_code | Latest promotion decision reason |
| promotion_promoted | boolean | yes | panel.promotion_latest_decision.promoted | If latest challenger was promoted |
| promotion_challenger_mae | number | yes | panel.promotion_latest_decision.challenger_metric_value | Challenger MAE in latest decision |
| promotion_champion_before | string | yes | panel.promotion_latest_decision.champion_before | Champion before decision |
| promotion_champion_after | string | yes | panel.promotion_latest_decision.champion_after | Champion after decision |
| drift_status | string | yes | panel.drift_status | Drift report status |
| drift_detected | boolean | yes | panel.drift_detected | Drift detection flag |
| drifted_features_count | integer | no | len(panel.drifted_features) | Count of drifted features |
| consistency_promotion_aligned | boolean | yes | panel.consistency_checks.promotion_aligned_to_registry | Promotion consistency check |
| consistency_drift_aligned | boolean | yes | panel.consistency_checks.drift_aligned_to_registry | Drift consistency check |
| alerts_total | integer | no | panel.alerts_total (default 0) | Total active alerts |
| alerts_critical | integer | no | panel.alerts_critical (default 0) | Critical alerts count |
| alerts_warn | integer | no | panel.alerts_warn (default 0) | Warning alerts count |
| alerts_info | integer | no | panel.alerts_info (default 0) | Informational alerts count |

## 4. Semantic Rules

- drifted_features_count is the canonical BI field for drift magnitude.
- alerts_total, alerts_critical, alerts_warn, alerts_info must be treated as non-negative integers.
- When source sections are missing, nullable fields may be empty; this is valid governed behavior.
- The dataset is snapshot-oriented, not event-oriented.

## 5. Recommended Power BI Model Mapping

- Table name: GovernancePanelLatest
- Grain: one row per latest export
- Suggested measures:
  - AlertSeverityIndex = alerts_critical * 3 + alerts_warn * 2 + alerts_info
  - ChampionMAE = MAX(champion_mae)
  - ChampionRMSE = MAX(champion_rmse)

## 6. Data Quality Checks for BI Refresh

At refresh time, validate:
1. CSV exists.
2. Required columns exist:
   - generated_at
   - champion_model
   - promotion_status
   - drift_status
   - alerts_total
3. Numeric columns parse without coercion errors:
   - champion_mae
   - champion_rmse
   - drifted_features_count
   - alerts_total
   - alerts_critical
   - alerts_warn
   - alerts_info

If checks fail, mark dataset as refresh warning and keep previous successful snapshot in report visuals.
