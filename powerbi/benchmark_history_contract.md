# Benchmark History Power BI Contract

## 1. Purpose

This document defines the semantic contract for Power BI consumption of the benchmark history export generated by the training pipeline.

Producer:
- src/artifacts.py
- function: save_powerbi_benchmark_export

Input source artifact:
- artifacts/benchmark_history.csv

Output artifact consumed by BI:
- artifacts/powerbi_benchmark_export_latest.csv

## 2. Refresh Contract

- The CSV is regenerated whenever the training pipeline completes artifact persistence.
- The file represents the latest full benchmark history available at export time.
- Expected cardinality: one row per recorded training run.

## 3. Schema Contract (CSV Columns)

| Column | Type | Nullable | Source | Description |
|---|---|---|---|---|
| run_timestamp | string | no | benchmark_history.timestamp | Technical run identifier in YYYYMMDD_HHMMSS format |
| run_datetime | datetime (ISO string) | yes | parsed from benchmark_history.timestamp | Analytical datetime for time-series visuals |
| model_name | string | yes | benchmark_history.model | Model identifier used in the run |
| features_used | string | yes | benchmark_history.features_used | Pipe-separated feature family list |
| train_rows | integer | yes | benchmark_history.train_rows | Number of training rows |
| validation_rows | integer | yes | benchmark_history.validation_rows | Number of validation rows |
| mae | number | yes | benchmark_history.MAE | Validation MAE |
| rmse | number | yes | benchmark_history.RMSE | Validation RMSE |
| promoted | boolean | no | normalized from benchmark_history.promoted_to_champion | Promotion outcome for the run; defaults to False when audit data is missing |
| promotion_reason_code | string | yes | benchmark_history.promotion_reason_code | Promotion decision reason code |
| champion_before | string | yes | benchmark_history.champion_before | Champion model before decision |
| champion_after | string | yes | benchmark_history.champion_after | Champion model after decision |
| challenger_mae | number | yes | benchmark_history.challenger_metric_value | Challenger MAE at decision time |
| champion_mae_at_decision | number | yes | benchmark_history.champion_metric_value | Champion MAE at decision time |
| absolute_improvement | number | yes | benchmark_history.absolute_improvement | Absolute MAE improvement versus champion |
| relative_improvement | number | yes | benchmark_history.relative_improvement | Relative MAE improvement versus champion |

## 4. Semantic Rules

- `run_timestamp` is the technical lineage key.
- `run_datetime` is the canonical time axis for Power BI visuals.
- `promoted` must be treated as a stable boolean field.
- Historical runs created before promotion audit existed may have null promotion detail fields, but `promoted` must still resolve to `False`.
- `mae` and `rmse` are the canonical performance trend metrics.
- This dataset is event-oriented, not snapshot-oriented.

## 5. Recommended Power BI Model Mapping

- Table name: BenchmarkHistory
- Grain: one row per training run
- Recommended visuals:
  - MAE trend over `run_datetime`
  - RMSE trend over `run_datetime`
  - Promotion outcome count by `promotion_reason_code`
  - Champion transition table using `champion_before` and `champion_after`

Suggested measures:
- LatestMAE = MAX(mae)
- LatestRMSE = MAX(rmse)
- PromotionCount = COUNTROWS(FILTER(BenchmarkHistory, BenchmarkHistory[promoted] = TRUE()))
- RejectionCount = COUNTROWS(FILTER(BenchmarkHistory, BenchmarkHistory[promoted] = FALSE()))

## 6. Data Quality Checks for BI Refresh

At refresh time, validate:
1. CSV exists.
2. Required columns exist:
   - run_timestamp
   - run_datetime
   - model_name
   - mae
   - rmse
   - promoted
3. Numeric columns parse without coercion errors:
   - train_rows
   - validation_rows
   - mae
   - rmse
   - challenger_mae
   - champion_mae_at_decision
   - absolute_improvement
   - relative_improvement
4. `promoted` parses as boolean without mixed text values.

If checks fail, mark dataset as refresh warning and keep previous successful benchmark visuals in the report.
